# Simple Client

The ***Simple Client*** is a tool designed to facilitate various Neo4j queries to retrieve information related to data provenance generated by the [DPDS](../README.md) tool.

## Provenance Queries

Here is a table summarizing the available provenance queries:

| Id  | Provenance Query                     | Input   | Output                                             |
| --- | ----------------------------------- | ------- | -------------------------------------------------- |
| PQ1 | All Transformations                 | $ğ·$       | Set of operations applied to $ğ·$ and the features they affect. |
| PQ2 | Why-provenance                       | $ğ‘‘_{ğ‘–\bold{a}}$    | The input data that influenced $ğ‘‘_{ğ‘–\bold{a}}$.            |
| PQ3 | How-provenance                       | $ğ‘‘_{ğ‘–\bold{a}}$     | The input data and the operations that created $ğ‘‘_{ğ‘–\bold{a}}$. |
| PQ4 | Dataset-level Feature Operation     | $ğ· _{âˆ—\bold{a}}$    | Set of operations that were applied to feature $\bold{a}$. |
| PQ5 | Record Operation                     | $ğ·_{ğ‘–âˆ—}$    | Set of operations that were applied to record $ğ·_{ğ‘–âˆ—}$. |
| PQ6 | Item-level Feature Operation         | $ğ‘‘_{ğ‘–\bold{a}}$     | Set of operations that were applied to $ğ‘‘_{ğ‘–\bold{a}}$.     |
| PQ7 | Feature Invalidation                 | $ğ·, \bold{a}$    | The operation that deleted the feature $\bold{a}$.        |
| PQ8 | Record Invalidation                  | $ğ·, ğ‘–$    | The operation that deleted the record $ğ·_{ğ‘–âˆ—}$.    |
| PQ9 | Item Invalidation                    | $ğ·, ğ‘–, \bold{a}$  | The operation that deleted the item $ğ‘‘_{ğ‘–\bold{a}}$.      |
| PQ10 | Item History                        | $ğ‘‘_{ğ‘–\bold{a}}$     | All the elements derived and that will derive from $ğ‘‘_{ğ‘–\bold{a}}$. |

| PQ13 | Impact on Dataset Spread            | $ğ·$       | The change in dataset spread of all operations applied to $ğ·$. |

## Usage

You can execute the ***Simple Client*** with different commands to perform specific queries. Below is an example of how to run the program for a specific command:

```bash
python main.py 
    --uri <URI>
    --user <USER>
    --pwd <PWD>
    <COMMAND>
```

> Before running the script, make sure you have properly configured the Python environment by following the instructions described in the [installation](../README.md#installation) section.

### Command Line Arguments

The following command line arguments are available for the ***Simple Client***:

- `<URI>`: The URI of the Neo4j server (default: `bolt://localhost`).
- `<USER>`: The username to access Neo4j (default: `neo4j`).
- `<PWD>`: The password to access Neo4j (default: `adminadmin`).
- `<COMMAND>`: One of the available commands:

  - `all-transformations`
  - `why-provenance`
  - `how-provenance`
  - `dataset-level-feature-operation`
  - `record-operation`
  - `record-invalidation`
  - `item-invalidation`
  - `item-level-feature-operation`
  - `item-history`
  - `record-history`
  - `feature-invalidation`
